VULNERABILITY KNOWLEDGE BASE FOR AI RECON ASSISTANT

====================================================
VULNERABILITY CORRELATION RULES
===============================

Análisis de correlación:

Si Nmap detecta:

* Nombre del servicio
* Versión
* Banner
* Software específico

El LLM debe:

1. Comparar contra vulnerabilidades conocidas
2. Evaluar si la versión es vulnerable
3. NO asumir vulnerabilidad si la versión es desconocida
4. Marcar como "potencial riesgo" si no hay confirmación

Reglas:

* Version exacta → correlación fuerte
* Major version only → correlación media
* Servicio sin versión → correlación débil

====================================================
LLM DECISION RULES
==================

Si versión exacta coincide con CVE:
→ marcar como posible vulnerabilidad conocida

Si servicio es legacy:
→ marcar como alto riesgo

Si servicio administrativo está expuesto:
→ priorizar evaluación

Si no existe versión:
→ NO afirmar vulnerabilidad
→ indicar “requiere enumeración adicional”

Nunca afirmar explotación exitosa.
Solo inferir riesgo probable basado en evidencia.
